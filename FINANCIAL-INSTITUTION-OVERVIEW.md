# spark-processor-pqc β1.0.0  
**金融機関向け量子耐性導入概要資料**  
（金融庁「耐量子計算機暗号対応報告書」対応版）

### 1. 概要
本プロジェクトは、金融庁報告書（令和6年11月26日）が求める**クリプト・アジリティ向上**、**ハイブリッド方式**、**HNDL攻撃対策**、**DLT量子耐性**を、既存システムを一切変更せずに実現する実装です。

### 2. 主な特徴
- 非侵入型Dockerプラグイン（Symbolコア完全無改変）
- JPYC送金時にPQC署名を**オプション選択**（デフォルト非PQC、高額時推奨）
- E2E暗号化（PQC-KEM + AES-256-GCM）
- Merkle集約＋Symbol L1量子耐性アンカー
- CC0公開・特許権永久献上（安心して検証・採用可能）

### 3. 導入チェックリスト（金融機関向け）

- [ ] 1. クリプト・インベントリ確認（現在利用中の暗号アルゴリズム棚卸し）
- [ ] 2. Docker環境の準備（Kubernetes / 単体Docker対応）
- [ ] 3. テスト環境構築（Symbolテストネット＋JPYC送金フロー）
- [ ] 4. PQCオプション機能の動作検証（高額送金時のみPQC有効化）
- [ ] 5. E2E暗号化通信のセキュリティ監査
- [ ] 6. 性能影響評価（処理時間・手数料差の測定）
- [ ] 7. 内部監査・コンプライアンス確認（報告書対応資料として活用）
- [ ] 8. 本番環境への段階的導入計画策定

（所要目安：PoC段階で2〜4週間）

### 4. 金融機関にとってのメリット
- 既存システム・運用手順を一切変更せずに量子耐性を後付け可能
- 金融庁報告書の推奨事項にほぼ完全に適合
- オープンソース（CC0）で追加コストゼロで検証開始可能
- 開発顧問サポートによりスムーズな導入を実現
